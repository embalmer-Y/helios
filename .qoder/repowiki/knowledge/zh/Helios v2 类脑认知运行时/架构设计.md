- **分层架构**：采用 Owner-Boundary（所有者边界）设计，将认知功能拆分为独立的语义所有者（如 `appraisal`, `memory`, `consciousness`），通过 `composition` 层进行组装。
- **核心引擎**：`RuntimeKernel` (`runtime/kernel.py`) 负责生命周期编排，按 `CANONICAL_STAGE_ORDER` 顺序执行阶段，并通过 `RuntimeFrame` 传递不可变状态。
- **契约驱动**：各模块通过 `contracts.py` 定义输入输出协议，`stages.py` 提供适配器将所有者 API 映射为运行时阶段。
- **依赖治理**：启动时通过 `dependencies.py` 执行关键依赖的 fail-fast 验证，确保无降级路径。
- **可观测性**：集成 `observability` 所有者，记录结构化日志并重建执行时间线，支持只读评估与因果链追溯。
