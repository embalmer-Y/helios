# Requirement 20 - Brain architecture comparison and scientific grounding

## 1. Task Breakdown

### Task 1 - 建立比较方法与映射表框架

1. 定义功能类比、工程替代、空缺和证据等级。
2. 搭建 Helios 域到脑功能系统的基础映射表。
3. 先覆盖核心域：情绪/调节、记忆、规划执行、主动性、自我模型、感知输入/行动输出。
4. 首个验证：检查映射表是否完整覆盖核心域。

### Task 2 - 补文献依据与差距分析

1. 为主要映射补充论文、综述或教材级依据。
2. 输出当前覆盖、主要缺口和不可比边界。
3. 将主要缺口映射回 requirement 编号。
4. 首个验证：人工审查每节是否都有来源类别与 requirement 回链。

### Task 3 - 接入 docs 总入口与 requirement 索引

1. 在 `docs/index.md` 中加入比较文档入口。
2. 在 `docs/requirements/index.md` 中说明 R20 的支撑作用。
3. 保持与哲学/HLD 文档的术语一致。
4. 首个验证：检查文档链接和术语一致性。

## 2. Dependencies

1. 依赖 R19 提供的 owner/boundary 文档基线。
2. 依赖现有哲学、HLD、roadmap 文档和模块审查矩阵。
3. 对外部专业论文和综述的引用需要在实际撰写时补齐。

## 5. Validation Plan

1. 检查每个核心域是否都有映射和证据等级。
2. 检查每个主要差距是否回链到 requirement。
3. 检查 docs 入口和 requirements index 链接完整。

## 6. Completion Criteria

1. 形成一份结构完整的科学比较文档方案。
2. 主要映射均有来源类别和差距说明。
3. R20 已纳入 docs 总入口与 requirements index。
4. 文档明确哪些类比只是启发，而不是实现宣称。
