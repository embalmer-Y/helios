推荐阅读顺序

如果你是第一次接手新版工程，按下面的顺序能最快建立正确的结构感。

Recommended Reading Order

If you are approaching the restructured codebase for the first time, this order gives the fastest path to a correct mental model.

1. 架构总览先读 `ARCHITECTURE.*`，明确当前分层和拥有权。
2. 详细设计再读 `DESIGN_PHILOSOPHY.*`，理解 tick 生命周期、对象流和设计约束。
3. 实现映射读 `IMPLEMENTATION_REFERENCE.*`，把模块、类与关键函数对应到理论和测试。
4. 资料目录读 `SOURCE_CATALOG.*`，确认仓库内资料、引用条目和待收集清单。
5. HTML 架构图打开 `architecture_overview.html`，把文本边界和可视化图对应起来。
6. 边界速查用 `current_structure.md` 做快速回看。
7. 理论研究最后再进入 DMN、neurochem、FEP 等 foundational 文档。
1. Architecture overviewRead `ARCHITECTURE.*` first to understand the current layers and ownership.
2. Detailed designRead `DESIGN_PHILOSOPHY.*` next for tick lifecycle, object flow, and design constraints.
3. Implementation referenceRead `IMPLEMENTATION_REFERENCE.*` to map modules, classes, and key methods back to theory and tests.
4. Source catalogRead `SOURCE_CATALOG.*` to inspect in-repo materials, citation entries, and collection backlog.
5. HTML system mapOpen `architecture_overview.html` to align the text model with the visual map.
6. Boundary quick sheetUse `current_structure.md` as a fast refresher.
7. Foundational notesOnly then move into DMN, neurochem, FEP, and related theory notes.
