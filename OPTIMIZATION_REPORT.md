## 二、问题分级

### 🔴 严重 — 影响核心体验

#### 1. JSON 解析脆弱 (42% 裸输出)

```
根因: LLM 输出的 JSON 含换行 → json.loads() 失败
      → raw text 被当 language_output 展示
      → 用户看到 `{"language_output":"..."}` 的丑陋格式

修复: regex 预处理去掉换行 + 多次回退策略
工作量: ~1h
```

#### 2. Φ 天花板 — 永远到不了心流/巅峰

```
根因: 5 源中 sensory_integration 被模拟源卡在 0.33
      → 加权平均无法突破 0.55

Φ 贡献拆解 (估算):
  感官整合:    ~0.33  ← 🚧 瓶颈！
  情感共振:    ~1.00  (活跃系统多时)
  DMN 深度:     ~0.40
  自我反思:    ~0.62
  点火增强:    ~0.55

修复方案A: 提高感官源权重或引入真实L0 (摄像头/麦克风)
修复方案B: 动态调整源权重 — 感官弱时其他源权重↑
修复方案C: 添加"元认知层" — 意识到自己天花板低也是一种意识
工作量: ~2h (方案B/C)
```

### 🟡 中等 — 影响系统深度

#### 3. 记忆巩固不产出

```
根因: MemoryConsolidator 聚类需要 ≥2 同标签情景
      → 事件多样 (23种) → 单标签最多1-2条 → 不聚类
      → 语义事实永远停在初始 3 个
      → 自传时刻阈值 (phi>0.25 & imp>0.3) 太严格

修复: 
  · 聚类改用粗粒度标签 (positive/negative/neutral 3类)
  · 自传阈值降到 phi>0.15 & imp>0.1
  · 添加"每日摘要"机制: 每 N 次巩固生成一条叙事
工作量: ~1h
```

#### 4. 情感回应趋同

```
根因: SEEKING 和 PLAY 长期主导 (>90% 时间)
      → DMN 20颗种子中正向占 14 颗
      → 负向情感 (FEAR/PANIC/RAGE) 几乎不激活
      → 回应变得"越来越甜"

数据:
  主导情感: SEEKING > PLAY > CARE > LUST
  几乎不出现: FEAR, RAGE

修复:
  · 长时运行引入"疲劳"机制 → 降低 SEEKING/PLAY 敏感度
  · 定期注入"挑战事件" → 强制激活负向系统
  · DMN 种子按运行时间动态调整权重
工作量: ~2h
```

#### 5. Φ "锁死"不回落

```
根因: 指数平滑 α=0.3 + 持续高情感共振
      → 一旦升到 0.5 就几乎不下滑
      → 失去了"清醒-迷糊"的动态呼吸感

修复:
  · 静息期 α 加倍 (0.3→0.6) — 让 Φ 更快回落
  · 添加"疲倦"因子: 长时间高 Φ 后强制下行
  · 清醒-迷糊波动更明显 (0.3-0.7 区间)
工作量: ~1h
```

### 🟢 轻微 — 锦上添花

#### 6. 记忆不持久化

```
现状: 情景记忆在内存，重启即丢失
目标: 定期序列化到磁盘 (每100条或每次巩固)
工作量: ~0.5h
```

#### 7. DMN 种子静态

```
现状: 20颗硬编码种子，永不变化
目标: 从情景记忆中动态提取新种子
工作量: ~3h (依赖记忆系统先完善)
```

#### 8. 缺少人格进化

```
现状: Helios 经过2000+次交互，人格参数不变
目标: 长期情感平均值缓慢调整人格基线
工作量: ~4h (需要先解决记忆+自传)
```
