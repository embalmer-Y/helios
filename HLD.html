Helios High-Level Architecture

这张图只保留真实运行主线的一级结构。它不试图展开每个类和函数，而是强调当前系统如何从事件进入认知核心，再流向语言、动作、记忆与外部输出，并最终形成反馈闭环。

Runtime Entry

编排器与周期控制

当前系统真正的运行入口是 demo 脚本，而不是统一 Core 类。它负责场景注入、节律切换、系统实例化、状态推进和 checkpoint。

Memory

记忆子系统

负责工作记忆、情景记忆、语义记忆和自传时间线，并在低 Φ 或静息阶段执行巩固与抽象。

Working / Episodic 保留短期上下文与带情感标签的经历。
Semantic / Autobiographical 把片段经验抽象成事实、模式与自我叙事。
Retriever / Consolidator 为 LLM 提供回忆上下文，并在静息期做跨层迁移。
Cognitive Core

认知主干

系统从事件进入神经化学状态，再经过驱动、情感、DMN 思维和统一 Φ 聚合，最终形成“当前意识态”的高层表征。

Neurochem + Drives 四种神经调质构成底层动力场；当前主线 demo 直接从它们合成 DriveVector。
Panksepp Emotions 7 大原始情感系统输出 AffectState，成为后续思考、记忆和决策的共同输入。
Thinking + UnifiedPhi DMN 产生思绪，UnifiedPhi 聚合感知、情感、思考、点火与自我模型，生成统一 Φ。
Language + Action

语言与执行

当点火条件满足时，系统调用 LLM 生成理解、独白与 decision，再通过 limb_decision_bridge 映射为安全动作。

LLM Layer 当前主线主要直接调用 OpenAI client，而不是统一走 llm_bridge。
Limb Layer 所有动作统一经由 HeliosBody、LimbRouter 和 Safety Rules 执行。
Feedback ActionResult 会回流到神经化学状态，形成 success / failure / blocked 的反馈闭环。
External Surface

外部输出与持久化

主线输出包括终端独白、journal 写入、checkpoint 存档、运行统计和自传叙事。这里是系统和外部观察者真正接触的界面层。

Not On The Hot Path

存在但不在当前主线热路径

这些模块对理解系统演化很重要，但它们不是当前 demo 里每个 cycle 的必经路径。

DriveOracle / HeliosSnapshot 更完整的驱动建模方案，当前主线没有直接把它当运行核心。
LLMBridge + llm_prompts 框架化封装仍然存在，但主线 demo 更偏向直接调用 API。
emotional_memory.py 兼容残留层，功能上已由新的 MemorySystem 接管。
archive / L0-L3 旧意识分层架构与历史 demo，更多承担理论来源与演化记录角色。

主干阅读顺序：编排器 → 认知主干 → 点火判定 → 语言与执行 → 外部输出；左侧记忆和底部旁路模块提供上下文与演化背景。
